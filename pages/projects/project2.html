ツイステ魔法石計算機 開発記事

概要

ツイステ魔法石計算機は、ツイステッドワンダーランド のガチャを引きたい日までにどれだけの魔法石を貯められるかをシミュレーションする Web アプリ。

現在の所持石・鍵、誕生日ボーナス、ショップ交換鍵、30daysパス、初回限定＆有償パックなどを総合的に計算し、目標の連数に届かない場合は最適な購入プランを提案する。

きっかけ

ツイステは友達に誘われて始めたゲームで、最初はあまり興味がなかったのですが、ストーリーやキャラクターにどんどん惹かれていきました。

しかし、ガチャを引くための魔法石をどれだけ貯めればいいのか、いつまでにどれだけの石が必要なのかが分からず、その度に電卓を叩いて計算していました。

そこで、ツイステのガチャを引きたい日までにどれだけの魔法石を貯められるかをシミュレーションする Web アプリを作ることにしました。

このアプリは、ツイステのガチャを引くための魔法石を効率的に貯めるためのツールとして、ツイステプレイヤーの皆さんに役立ててもらえることを願っています。

開発環境とフレームワーク

  • JavaScript + React（Function Components + Hooks）
  • Vite
  • Cloudflare Workersでデプロイ

こだわったポイント

  • 全部 useState で状態を管理する。
  • 計算は副作用がない関数にして、テストしやすくする。
  • 入力と結果の画面をカテゴリごとに分けてコンポーネントにする。
  • 購入プランは再帰探索＋枝刈りで計算。

ログインボーナスやミッションで獲得できる魔法石

ショップで交換できる鍵

今月分：配布鍵3個、ショップ（10連鍵+鍵3個）実質90魔石、イベント鍵3個（オプション）
来月以降：月ごとの合計魔石価値を月数×（配布90+ショップ90+（イベント90））で算出

30daysパスの計算

function calculate30DaysPassActualStones(count, daysUntilTarget) {
  let total = 0;
  for (let i = 0; i < count; i++) {
    const offset = i * 30;
    if (offset < daysUntilTarget) {
      total += 48; // 購入即時
      total += Math.min(daysUntilTarget - offset, 30) * 5;
    }
  }
  return total;
}

購入提案アルゴリズム

初回限定パック、30daysパス、1日限定パック、通常パックを効率（魔法石/円）順にソートし、
再帰的に「何個買うか」を全探索＋枝刈りし、必要石数を満たす最安組み合わせを出力。
