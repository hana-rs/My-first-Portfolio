React × TypeScript ルールブック（用語解説付き）

1. 基本の考え方

🔤 用語

JavaScript (JS)
Web の動きを作るプログラミング言語。ボタンを押したときに何かが起きるのも JS。

TypeScript (TS)
JavaScript に「型」というルールを加えて、安全にした言語。→ バグを減らせる。

React
UI（画面）を部品（コンポーネント）として組み立てるライブラリ。→ 「小さい部品を組み合わせて画面を作る」イメージ。

2. コンポーネント

🔤 用語

コンポーネント = 「画面の部品」

Props = コンポーネントに渡すデータ（材料）

✅ コード例

<!-- Props（材料）の型 -->
type Props = {
  title: string
}

<!-- 関数コンポーネント -->
const Header = ({ title }: Props) => {
  return <h1>{title}</h1>
}

<!-- 使う側 -->
<Header title="こんにちは" />

📝 解説

<Header title="こんにちは" /> で文字を渡している。Props を型で書くから「何を渡せるか」がすぐわかる。

3. useState（状態）

🔤 用語

状態（state） コンポーネントの中で変化するデータ。

Hook React が用意した便利関数（useState, useEffect など）。

✅ コード例

import { useState } from "react"

const Counter = () => {
  const [count, setCount] = useState<number>(0)
  // count = 今の数
  // setCount = 数を更新する関数

  return (
    <div>
      <p>カウント: {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  )
}

📝 解説

useState(0) → 初期値は 0。count が今の数、setCount が数を変える関数。値を変えると画面も再描画される。

4. イベント

🔤 用語

イベント = ユーザー操作（クリック・入力など）

イベントハンドラ = イベントが起きたときに実行される関数

✅ コード例

const handleClick = (e: React.MouseEvent<HTMLButtonElement>) => {
  alert("クリックされた！")
}

<button onClick={handleClick}>クリックしてね</button>

📝 解説

onClick に関数を渡すとクリック時に実行される。e は「イベントの情報」。

5. フォーム部品

🔤 用語

テキストボックス = 文字を入力する欄

制御されたコンポーネント = 入力値を state で管理する仕組み

✅ コード例（テキストボックス）

const [text, setText] = useState<string>("")

<input
  type="text"
  value={text}
  onChange={e => setText(e.target.value)}
  placeholder="ここに入力"
/>

<p>入力中: {text}</p>

📝 解説

value={text} → 入力欄と state が同期する。onChange で更新。placeholder は入力前の説明。

✅ コード例（セレクトボックス）

const [fruit, setFruit] = useState<string>("apple")

<select value={fruit} onChange={e => setFruit(e.target.value)}>
  <option value="apple">りんご</option>
  <option value="banana">バナナ</option>
</select>

✅ コード例（チェックボックス）

const [checked, setChecked] = useState<boolean>(false)

<input
  type="checkbox"
  checked={checked}
  onChange={e => setChecked(e.target.checked)}
/>

6. リスト表示

🔤 用語

配列（Array） = データを並べて持つ形

map = 配列の中身を一つずつ処理する関数

✅ コード例

const fruits = ["りんご", "バナナ", "みかん"]

<ul>
  {fruits.map((fruit, i) => (
    <li key={i}>{fruit}</li>
  ))}
</ul>

📝 解説

map で配列を <li> に変換。key は React が管理のために必要。

7. useEffect（副作用）

🔤 用語

副作用（Effect） = 画面描画以外でやる処理（API呼び出し、タイマーなど）

✅ コード例

const [text, setText] = useState<string>("")

// 初回だけ実行
useEffect(() => {
  console.log("初回だけ")
}, [])

// text が変わるたびに実行
useEffect(() => {
  console.log("入力された:", text)
}, [text])

📝 解説

[] = 初回だけ、[text] = text が変わるたび

8. 子コンポーネントにデータを渡す

🔤 用語

Props = 子コンポーネントに渡すデータ

✅ コード例

type ChildProps = { message: string }

const Child = ({ message }: ChildProps) => <p>{message}</p>

const Parent = () => <Child message="こんにちは" />

📝 解説

親 → 子に「データ」を渡せる。型を決めておくと安全。

9. よく出る単語まとめ

  • コンポーネント = 部品
  • Props = 部品に渡すデータ
  • State = 変化するデータ
  • Event = ユーザー操作
  • Hook = React の便利関数 (useState, useEffect)
  • map = 配列を繰り返し処理
